# Usage des pages

## Objectif

Ce document sert de mode d'emploi concret de la webapp interne.
Il ne décrit pas l'architecture; il décrit comment utiliser chaque page au quotidien.

## Pages principales

### `/dashboard`

Usage :
- voir l'état rapide des bureaux déjà validés
- repérer les bureaux les plus mobilisés
- garder en tête quelques indicateurs INSEE utiles

À utiliser pour :
- l'ouverture d'une réunion
- un point rapide de pilotage
- vérifier que la base locale remonte bien des données

### `/search`

Usage :
- retrouver vite un citoyen, un retour terrain ou une tâche
- éviter de naviguer module par module

À utiliser pour :
- chercher un nom, un téléphone, un sujet ou un mot-clé
- vérifier si un signal terrain existe déjà
- retrouver une tâche liée à un retour

### `/polling-stations`

Usage :
- consulter les 9 bureaux de vote
- voir leur lieu, adresse et statut
- visualiser les bureaux validés sur la carte

À utiliser pour :
- préparer le terrain
- expliquer la géographie électorale à l'équipe
- vérifier qu'un retour ou une fiche est rattaché au bon bureau

### `/electoral-analysis`

Usage :
- lire les résultats municipaux 2026 T1 déjà validés
- comparer les bureaux entre eux
- croiser signaux terrain et lecture électorale

À utiliser pour :
- repérer les bureaux sensibles
- prioriser un secteur à traiter
- préparer une note politique rapide

### `/insee`

Usage :
- consulter les grands repères socio-démographiques de Cabestany

À utiliser pour :
- contextualiser un quartier ou un bureau
- préparer un argumentaire
- recouper les données électorales avec la sociologie locale

### `/tasks`

Usage :
- créer, assigner et suivre les tâches
- transformer un besoin opérationnel en action

À utiliser pour :
- suivre l'exécution de campagne
- affecter un responsable
- relancer les tâches bloquées

### `/meetings`

Usage :
- créer des réunions
- stocker notes et actions liées

À utiliser pour :
- garder une trace des décisions
- faire ressortir les suites concrètes d'une réunion

### `/field-reports`

Usage :
- saisir les remontées terrain
- qualifier un signal
- rattacher le retour à un citoyen, un bureau, une priorité

À utiliser pour :
- faire remonter les irritants locaux
- garder la mémoire du terrain
- créer une tâche depuis un retour prioritaire

### `/field-analysis`

Usage :
- lire les tendances terrain déjà saisies
- voir les thèmes dominants, urgences et bureaux les plus chargés

À utiliser pour :
- préparer le terrain de la semaine
- faire remonter les signaux politiques
- identifier les urgences ouvertes

### `/citizens`

Usage :
- gérer les fiches habitants
- voir le soutien, les tags et le bureau
- consulter la timeline de retours et de tâches liées

À utiliser pour :
- suivre les contacts importants
- préparer une relance ciblée
- centraliser les informations utiles sur une personne

### `/team`

Usage :
- piloter la couverture des secteurs
- affecter un responsable par secteur
- voir les priorités d'action

À utiliser pour :
- répartir le terrain
- repérer les trous de couverture
- arbitrer où agir d'abord

### `/users`

Usage :
- créer et gérer les comptes
- attribuer rôles, fonctions et permissions

À utiliser pour :
- ouvrir l'accès à un nouveau membre
- corriger un périmètre d'accès
- réinitialiser un mot de passe

## Rappel

- `superadmin` : accès total
- les pages apparaissent selon les permissions réelles
- la base locale PostgreSQL fait foi pour les données de travail
